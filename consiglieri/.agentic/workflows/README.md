# Don Consiglieri Workflows

This directory contains executable example workflows for the **Don Consiglieri** agentic orchestration framework.

Workflows in this directory are discovered dynamically by the TUI console application and run as standalone processes via `go run`.

---

## Workflow Discovery Metadata

The TUI discovers workflows using the [DiscoverWorkflows](file:///home/igor/Documents/projetos/don/consiglieri/internal/tui/workflow_list.go#L24) parser. To ensure a workflow is properly listed and styled in the dashboard, include the following single-line comments in your source file:

```go
// name: <Workflow Display Name>
// description: <Brief explanation of what the workflow does>
```

If the `// name:` comment is omitted, the name will default to the Go source file's base name (e.g., `hello` for `hello.go`).

## Example Workflows

### 1. Hello Workflow
*   **File**: [hello.go](file:///home/igor/Documents/projetos/don/consiglieri/.agentic/workflows/hello.go)
*   **Purpose**: Demonstrates a basic successful execution of a pipeline.
*   **Behavior**: It instantiates a simple [primitives.Pipeline](file:///home/igor/Documents/projetos/don/consiglieri/pkg/primitives/pipeline.go) containing a loop that performs mock work for 5 steps (sleeping 500ms between each) and returns `nil` to indicate successful completion.

### 2. Force Error Workflow
*   **File**: [force_error.go](file:///home/igor/Documents/projetos/don/consiglieri/.agentic/workflows/force_error.go)
*   **Purpose**: Demonstrates error handling, process failures, and reporting.
*   **Behavior**: Similar to the Hello Workflow, it runs a loop for 5 steps. However, once the loop completes, it deliberately returns a forced error (`errors.New("Something bad happened")`). The orchestrator catches this error and shuts down the session with a failure state, which is visually captured by the TUI Dashboard's execution view.

### 3. Try Agentic Workflow
*   **File**: [try_agentic.go](file:///home/igor/Documents/projetos/don/consiglieri/.agentic/workflows/try_agentic.go)
*   **Purpose**: Demonstrates LLM agent execution within a workflow pipeline.
*   **Behavior**:
    *   Creates a [primitives.Agent](file:///home/igor/Documents/projetos/don/consiglieri/pkg/primitives/agent.go) configured to use [primitives.AgyProvider](file:///home/igor/Documents/projetos/don/consiglieri/pkg/primitives/AgentProvider.go).
    *   Instructs the agent with the prompt: `"Explain what is simple Agent in 50 lines."` using the `gemini-3.1-pro` model.
    *   Wraps the agent in a pipeline. When run, it prints the location of the resulting artifact path generated by the agent.

### 4. Joke History Workflow
*   **File**: [joke_history.go](file:///home/igor/Documents/projetos/don/consiglieri/.agentic/workflows/joke_history.go)
*   **Purpose**: Demonstrates a multi-step sequential workflow where outputs from previous agent steps are passed as context to subsequent steps.
*   **Behavior**:
    *   Runs **Agent A** to generate a funny joke (max 5 lines).
    *   Runs **Agent B** to generate another funny joke (max 5 lines).
    *   Runs **Agent C** to write a short story (max 50 lines) that incorporates both jokes.

## Execution Instructions

You can launch and monitor these workflows in two ways:

### 1. Interactive Dashboard (TUI)
To run these workflows interactively via the Consiglieri Dashboard:
1. Navigate to the `consiglieri` directory.
2. Build or run the CLI:
   ```bash
   go run cmd/cli/main.go
   ```
3. Use the arrow keys to navigate the workflow list, select the desired workflow (e.g., **Hello Workflow**, **Force Error Workflow**, or **Try Agentic Workflow**), and press `Enter` to run it. The console will display real-time execution logs.

### 2. Standalone CLI Run
Because each workflow is structured as a `package main` Go entrypoint, you can run them directly using the Go toolchain from the `consiglieri` directory:
```bash
go run .agentic/workflows/hello.go
go run .agentic/workflows/force_error.go
go run .agentic/workflows/try_agentic.go
go run .agentic/workflows/joke_history.go
```

## Guide to Creating New Workflows

To add a new workflow:

1.  **Create a Go file** in `.agentic/workflows/` (e.g., `.agentic/workflows/my_workflow.go`).
2.  **Define metadata comments** at the top of the file so that the TUI can discover it:
    ```go
    // name: My Custom Workflow
    // description: A pipeline that runs my custom tasks.
    package main
    ```
3.  **Import the primitives package**: `"don_consiglieri/pkg/primitives"`
4.  **Create Pipelines or Agents** in the `main` function:
    ```go
    p := primitives.NewPipeline("My Pipeline", func(ctx context.Context) error {
        // Your custom logic or agent execution here...
        return nil
    })
    ```
5.  **Add the pipeline to an Orchestrator** and run it:
    ```go
    orch := primitives.NewOrchestrator("My Orchestrator")
    orch.AddWorkflow(p)

    if err := orch.Run(context.Background()); err != nil {
        log.Fatalf("Orchestrator failed: %v", err)
    }
    ```
